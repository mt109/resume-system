# Google PM patterns

Distilled from resume sessions. Format: **Stop / Continue / Start**.

---

## Stop

- **Stop writing "what you shipped" bullets for Google applications.** Google screens for "what changed" — user impact, business outcome, behavior shift. Feature names alone don't pass.
- **Stop leaving experimentation implicit.** If you don't have an explicit bullet about hypothesis-driven testing, metrics definition, or experiment evaluation, add one. Experimentation is table stakes for Google PM roles.
- **Stop hiding ML exposure.** Even without model ownership, coursework (TensorFlow, PyTorch), research (LSTM, Neural Prophet), and product decisions informed by ML signals all count. Silence reads as "no ML experience."
- **Stop using Offer Wallet numbers externally without compliance review.** $76M/$69M, 500K+, 6.5M+, ~2× redemption are strong — but confirm with Microsoft what's cleared for external use before every submission.

## Continue

- **Continue framing ML as a partnership.** "My strengths are problem framing, metrics, experiment design, and cross-functional execution — not pretending I run a global recsys." Google interviewers respect this framing over overclaiming.
- **Continue surfacing "Googleyness" signals naturally.** Intellectual humility ("I describe scope honestly"), learning agility (Claude Code course, cross-domain ramp), comfort with ambiguity (identity/entitlement architecture), and community impact (Digital Literacy, DEI, alumni leadership) all read well at Google.
- **Continue using the Offer Wallet story as the strongest quantitative proof point.** ~2× engagement, campaign velocity doubling, 500K+ net-new reach, $76M/$69M projection — this is the best "what changed" narrative in the portfolio (with scope caveats).

## Start

- **Start adding an explicit experimentation bullet to every Google-targeted resume.** Even lightly framed: "Designed and evaluated product experiments to improve content engagement and subscription conversion."
- **Start positioning Xbox work as "content surfacing and discovery" for discovery-adjacent roles.** The same facts — what appears, where, to whom — map directly to recommendation and ranking problems when framed correctly.
- **Start building a YouTube/gaming discovery teardown or mini-prototype** to close the ML/recsys gap. Analyze "why am I seeing this video?" or build a simple ranker — this moves from "competitive" to "top 5% candidate."
